How I'm Learning

I've spent some time researching the best way to take notes. I ended up finding a tool called Quiver that I'm currently trialing, but it seems to be really great for documentation.

I spent some time playing around with it to decide how to actually setup my documentation. I begrudgingly ended up reading the tutorial, which was actually quick and very helpful. I then landed on the following approach, using a tool called Quiver.

Method:
  • Use markdown to explain what I'm doing
  • Insert an example of the code in actions
  • Show the code

This method allows me to take detailed notes that are easily readanble using Quiver's preview functionality. When I review the notes in the future, I can quickly use the preview view to find my codes and see what they do.

Benefits of this Method:
  1. I have detailed documentation to reference
  2. My documentation can be shared with others.
  3. I learn better by recording everything I do.
